# Sales-Store-Data-Analysis-using-SQL
## 📌 Project Overview

This project focuses on analyzing retail sales data using SQL to uncover meaningful business insights. The dataset includes customer details, product information, transaction records, and purchase behavior.

The analysis covers:

- Data cleaning & preprocessing
- Customer behavior analysis
- Product performance evaluation
- Revenue and sales trend analysis

The goal is to transform raw transactional data into actionable insights that can drive data-driven decision-making.

## 🎯 Problem Statement

Retail businesses often struggle with:

- Identifying high-performing products
- Understanding customer purchase behavior
- Detecting revenue leakage due to cancellations/returns
- Optimizing inventory and marketing strategies

This project aims to solve these problems by leveraging SQL to extract insights from sales data.

## 🛠️ Tech Stack
Database: PostgreSQL / SQL

Tools Used: SQL Editor PgAdmin
  
Techniques:
  - Data Cleaning
  - Aggregations
  - Window Functions
  - CASE Statements
  - Joins & Grouping

## 🧹 Data Cleaning & Preparation

Key steps performed before analysis:

- Removed duplicate records using DISTINCT and ROW_NUMBER()
- Handled missing values and inconsistent entries
- Standardized categorical data (e.g., M → Male, CC → Credit Card)
- Renamed incorrect column names (prce → price, quantiy → quantity)
